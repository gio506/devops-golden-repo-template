# Files Explained

- `.editorconfig`: formatting defaults shared across editors.
- `.gitignore`: safe ignore rules for common tooling and generated files.
- `.markdownlint.json`: markdown lint policy.
- `.pre-commit-config.yaml`: optional local hooks for markdown and YAML checks.
- `.yamllint.yml`: YAML lint policy.
- `.github/PULL_REQUEST_TEMPLATE.md`: PR checklist aligned with the `dev -> main` workflow.
- `.github/workflows/ci-reusable.yml`: reusable standards CI for repo hygiene and safety checks.
- `.github/workflows/ci.yml`: caller workflow so the template validates itself.
- `CHEATSHEET.md`: quick maintainer command reference.
- `CODEOWNERS`: review ownership mapping.
- `CONTRIBUTING.md`: contributor process and expectations.
- `FILES_EXPLAINED.md`: file-by-file purpose guide.
- `LICENSE`: usage and distribution terms.
- `Makefile`: local quality commands.
- `README.md`: template overview and guidance.
- `SECURITY.md`: vulnerability reporting policy.
- `docs/standards.md`: branch, PR, and CI standards.
- `docs/template-checklist.md`: practical setup checklist for new repos.
